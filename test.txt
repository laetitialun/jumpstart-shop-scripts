LEXIQUE
----------
git status permet de voir les fichiers non poussés
mkdir => Créer un dossier
git diff => affiche ce qui a été modifier dans un fichier
git rm => supprimer un fichier
git add / git add . => création ou modification de fichers
git log => liste tous les commits
	git log -1
	git log -1 --stat
	git log -2 -- author "laetitialun"

git branch liste les branche en local
git rebase => permet de récupérer les modifications d'une autre branche en récupérant l'historique des commits.
git branch -d NOM DE LA BRANCH

POUSSER SON CODE VERS GITHUB [ADD + COMMIT + PUSH]
-----------------------------------------------------
Dans le fichier
1. Modifier le fichier

Dans le terminal
1. Vérifier le statut => 
	a. git status
	b. git diff NOM-DU-FICHIER.EXT
2. Ajouter le fichier => 
	a. git add NOM-DU-FICHIER.EXT
	b. git add .
3. Commiter avec un commentaire à propos du commit => git commit -m "COMMENTAIRE"
4. Pousser/Envoyer le commit vers GitHub => git push
